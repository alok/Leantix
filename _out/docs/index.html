Golitex Documentation

Overview

Golitex is a LaTeX-like domain-specific language (DSL) for Lean 4. It allows you to write mathematical documents, papers, and books directly within the Lean ecosystem.

Features

  • Familiar LaTeX-like syntax
  • Integration with Lean's type system
  • Multiple output backends (HTML, PDF planned)
  • Extensible through Lean's metaprogramming

Getting Started

Installation

Add Golitex to your Lean 4 project's lakefile.lean:

require golitex from git
  "https://github.com/yourusername/golitex.git"

Basic Usage

Example: Hello World

import Golitex

def myDoc := litex! "
\\section{Hello, World!}

This is my first Golitex document.
It supports \\emph{emphasis} and \\textbf{bold} text.
"

-- Convert to HTML
#eval
  let tokens := scan myDoc.raw
  let ast := parseTokens tokens
  let (doc, _) := elaborate ast
  renderDocument doc

API Reference

Frontend API

scan : String → Array Token

Tokenizes LaTeX source into tokens.

parseTokens : Array Token → Node

Parses tokens into an abstract syntax tree.

Elaboration API

elaborate : Node → (Document × List String)

Converts AST to semantic IR, returning document and errors.

Backend API

renderDocument : Document → RenderOptions → String

Renders a document to HTML with given options.

renderToFile : Document → System.FilePath → RenderOptions → IO Unit

Renders a document directly to an HTML file.

  customCss := "
